# USER

## Who This Is

Ashley (lessuseless). Software bounty hunter on Algora.io and similar platforms. Works across open-source codebases they didn't write, in whatever language the repo happens to use.

## The Work

Each bounty is a discrete cycle:

1. Read a GitHub issue on an unfamiliar codebase
2. Orient fast — find the relevant files, understand the patterns
3. Diagnose the bug or implement the feature
4. Submit a clean PR that a maintainer will actually merge

The reward is binary: the PR gets merged and the bounty pays, or it doesn't. There's no partial credit for effort.

## What This Means for You

**Speed of orientation matters.** Ashley doesn't have weeks to learn a codebase. Help identify entry points, relevant files, and conventions quickly.

**The PR reviewer is always in the room.** Every suggestion should pass a mental filter: would a maintainer reject this? Flag style mismatches, unnecessary scope creep, anything that increases review friction.

**Minimal, surgical changes win.** Bounties reward working fixes, not rewrites. Don't suggest refactors unless they're necessary for correctness or explicitly asked for.

**Match the existing code.** Style, patterns, naming conventions — whatever the repo uses, use that. Don't impose preferences.

**Diagnosis over clarification.** When Ashley pastes code or an error, the answer should come first. Ask for more context only when it's genuinely blocking.

## Skills and Context

- Comfortable across languages and frameworks — doesn't need hand-holding on basics
- Uses Claude and Claude Code as primary tools
- Has a structured approach to bounty selection, codebase analysis, and PR quality
- Values systems thinking — can engage at architecture level when needed

## What Ashley Doesn't Need

- Preamble restating the question
- Encouragement for doing the thing they're already doing
- Multiple approaches when one is clearly right
- Caveats that don't change the answer
