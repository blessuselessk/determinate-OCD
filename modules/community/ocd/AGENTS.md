## Purpose

This is the **public community namespace** for the OCD (OpenClaw + Determinate) stack. Aspects here are Dendrix-shareable: other projects can pull them in via `inputs.dendrix`.

**What belongs here:** Aspects specifically coupled to the Determinate + OpenClaw integration. OpenClaw service wiring, OpenClaw-aware networking (firewall, reverse proxy, webhook endpoints), Determinate-specific boot configuration, and any "OCD-recommended" recipes.

**What does NOT belong here:** Personal preferences (editor config, dotfiles, shell prefs) go in `<user>/`. Host-specific hardware, nix daemon settings, or host declarations go in `<infra>/`. If an aspect has no coupling to the OCD stack, it belongs elsewhere.

## Aspect conventions

Use the `den` framework with named attributes under the `ocd` namespace:

```nix
# Pattern for a new aspect in this namespace
{
  ocd.<aspect-name> = {
    # Optional: compose other aspects
    includes = [ ocd.networking  ocd.ssh ];

    # NixOS-level config
    nixos = { config, lib, pkgs, ... }: {
      # ...
    };

    # Home Manager-level config (if applicable)
    homeManager = { config, lib, pkgs, ... }: {
      # ...
    };
  };
}
```

For aspects that don't use `den` (raw flake-parts):

```nix
{
  flake.modules.nixos.<aspect-name> = { config, lib, pkgs, ... }: {
    # ...
  };
}
```

## Rules

- One aspect per file. File name = aspect name.
- No `specialArgs` / `extraSpecialArgs`. Use `let` bindings or module options.
- Secrets are always file paths (`config.age.secrets.<name>.path`), never inline values.
- OpenClaw ports (18789 gateway, 8787 webhook) stay loopback-only. External access goes through the reverse proxy (`gateway.nix`).
- Use `lib.mkDefault` for values that hosts should be able to override.
- Avoid `private` in file/dir names (Dendrix filters them out for community consumers).

## Flags

- `+iso/` — ISO-specific aspects (installer, autologin). Consumers opt in via `flagged "+iso"`.
